# Practice Guide: Chapter 1 (Volume 5)

## Objective
To understand how an AWS Lambda function receives and processes event payloads in Python.

## Assignment 1: The Event Payload
When AWS Lambda is triggered (for example, by a file being uploaded to S3), it doesn't just run blindly. AWS passes a JSON dictionary to the Python script called the `event`.

1. Review this theoretical JSON payload generated by an S3 upload:

    ```json
    {
      "Records": [
        {
          "eventSource": "aws:s3",
          "eventName": "ObjectCreated:Put",
          "s3": {
            "bucket": {
              "name": "company-raw-images-bucket"
            },
            "object": {
              "key": "vacation-photo.jpg",
              "size": 1024553
            }
          }
        }
      ]
    }
    ```

2. **Analysis:** The Lambda function needs to parse this JSON to figure out *which* bucket the file was uploaded to, and *what* the file's name (`key`) is.

## Assignment 2: The Lambda Handler
Every Lambda function must have an entry point, usually a Python function called `lambda_handler(event, context)`.

1. Open a new file: `nano lambda_function.py`

2. Write the theoretical Python code to parse the event and process the image:

    ```python
    import json
    import urllib.parse
    import boto3

    print("Loading function")

    # Initialize the S3 client
    s3 = boto3.client('s3')

    def lambda_handler(event, context):
        # 1. Parse the Bucket Name from the JSON
        bucket = event['Records'][0]['s3']['bucket']['name']
        
        # 2. Parse the File Name (Key) from the JSON
        key = urllib.parse.unquote_plus(event['Records'][0]['s3']['object']['key'], encoding='utf-8')
        
        try:
            print(f"Event received! A user uploaded file: {key} into bucket: {bucket}")
            
            # (Theoretical Code) Download the image from S3
            # (Theoretical Code) Use a library like Pillow to resize it to 200x200
            # (Theoretical Code) Upload the thumbnail back to a different S3 bucket
            
            return {
                'statusCode': 200,
                'body': json.dumps(f"Successfully processed {key}")
            }
        except Exception as e:
            print(e)
            print(f"Error processing object {key} from bucket {bucket}.")
            raise e
    ```

## Assignment 3: The Execution Flow

1. **Analysis:** When the user uploads `vacation-photo.jpg`, AWS spins up a Docker container and executes this Python script.

2. The `event` dictionary is passed into `lambda_handler`.

3. The script extracts `company-raw-images-bucket` and `vacation-photo.jpg`.
4. It prints a log message to AWS CloudWatch (this is how you troubleshoot Lambda!).
5. It performs the theoretical image processing and returns a `200 OK` status code. The container is then destroyed.
