# Practice Guide: Chapter 10 (Volume 3)

## Objective
To create a Logical backup using `mysqldump`, purposely destroy the database, and prove you can restore it from the `.sql` file.

## Assignment 1: The Setup
Let's add some data to the `webstore_db` we created in the last chapter.

1. Log into MariaDB as the `root` user:
   `mysql -u root -p`
2. Switch to the webstore database:
   `USE webstore_db;`
3. Create a simple table and insert a row of data:
   `CREATE TABLE products (id INT, name VARCHAR(50));`
   `INSERT INTO products VALUES (1, 'Linux Handbook');`
4. Verify the data exists:
   `SELECT * FROM products;`
5. Exit the database:
   `exit;`

## Assignment 2: The Backup
Now we will use the `mysqldump` tool to create our Logical Backup.

1. Run the dump command, redirecting the output (`>`) into a file in your home directory:
   `mysqldump -u root -p webstore_db > ~/webstore_backup.sql`
2. Let's look at the backup file. Use the `less` or `cat` command:
   `cat ~/webstore_backup.sql`
3. **Observation:** Look closely at the text! You will see the exact `CREATE TABLE` and `INSERT INTO` commands you typed earlier, automatically generated by the tool!

## Assignment 3: The Disaster
Oh no! It's 2:00 AM and you made a typo!

1. Log back into MariaDB:
   `mysql -u root -p`
2. Destroy the entire database:
   `DROP DATABASE webstore_db;`
3. View your databases to confirm it is gone forever:
   `SHOW DATABASES;`
4. Exit the database:
   `exit;`

## Assignment 4: The Restoration
Time to be the hero. 

1. Before we can restore the data, we must recreate the empty container for it. Log back into MariaDB:
   `mysql -u root -p -e "CREATE DATABASE webstore_db;"`
   *(The `-e` flag lets you execute a command without entering the interactive shell!)*
2. Now, use the input redirect (`<`) to blast your `.sql` file back into the database:
   `mysql -u root -p webstore_db < ~/webstore_backup.sql`
3. Log in one final time and check the data:
   `mysql -u root -p`
   `USE webstore_db;`
   `SELECT * FROM products;`
4. **Result:** The `Linux Handbook` product is back! You successfully restored the database from a total failure!

## Success Criteria
You have successfully completed this practice if you used `mysqldump` to create a `.sql` file, completely dropped your database, and successfully resurrected the `products` table using the `<` redirect operator.
