# Glossary & Index

## A
* **Auto-Scaling Group (ASG):** A cloud resource that automatically increases or decreases the number of compute instances based on defined metrics (e.g., CPU utilization).
* **Availability Zone (AZ):** One or more discrete data centers with redundant power, networking, and connectivity in an AWS Region.

## B
* **Boto3:** The official Amazon Web Services (AWS) SDK for Python, allowing engineers to write scripts that interact directly with the AWS REST API.

## C
* **CI/CD:** Continuous Integration and Continuous Deployment. The methodology of automatically testing, building, and deploying code using a pipeline.
* **Control Plane:** The components of an orchestration system (like Kubernetes) that make global decisions about the cluster (e.g., scheduling pods, detecting node failures).

## D
* **Distributed Tracing:** A method used to profile and monitor applications, especially those built using a microservices architecture, by tracking a single request using a unique Trace ID as it jumps across multiple services.

## E
* **Error Budget:** A metric used in Site Reliability Engineering (SRE) that defines the maximum amount of time a system is allowed to be down or in an error state before feature deployments must be frozen.

## I
* **IAM (Identity and Access Management):** The framework used to ensure that the right entities (users or machines) have the right access to resources.
* **IDP (Internal Developer Portal):** A self-service portal built by Platform Engineers that allows developers to provision cloud resources and deploy code without submitting IT tickets.

## K
* **Kubernetes (K8s):** An open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications.

## P
* **Prometheus:** An open-source systems monitoring and alerting toolkit built around a time-series database and a "pull" metric collection model.
* **PromQL:** Prometheus Query Language.

## S
* **SLI (Service Level Indicator):** A carefully defined quantitative measure of some aspect of the level of service that is provided (e.g., HTTP 200 responses as a percentage of total responses).
* **SLO (Service Level Objective):** A target value or range of values for a service level that is measured by an SLI.
* **SRE (Site Reliability Engineering):** A discipline that incorporates aspects of software engineering and applies them to infrastructure and operations problems.

## T
* **Toil:** In SRE, toil is the kind of work tied to running a production service that tends to be manual, repetitive, automatable, tactical, devoid of enduring value, and that scales linearly as a service grows.
* **Transit Gateway:** A cloud network transit hub used to interconnect virtual private clouds (VPCs) and on-premises networks.
